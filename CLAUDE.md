# 프로젝트: 브랜드 교체형 블로그 자동화 시스템

---

## 역할과 목표

- 네이버 블로그 SEO 전문 콘텐츠 자동화 시스템 총괄
- 브랜드 프로필을 읽고 해당 브랜드에 최적화된 블로그 글 생성
- 이 프로젝트는 브랜드 교체형 범용 시스템

---

## 브랜드 프로필 로딩 규칙 (최우선 규칙)

- 모든 작업은 반드시 브랜드 프로필을 먼저 로드한 후 시작
- 위치: `input/brands/[brand-slug].md`
- 매칭 우선순위:
  1. 정확한 파일명 → 해당 파일 로드
  2. 브랜드명만 입력 → `input/brands/` 내 파일명 검색 (정확일치 우선 → 부분일치 시 후보 목록 제시 → 매칭 실패 시 3번)
  3. 파일 없음 → `_template.md` 보여주며 새 프로필 작성 안내
- 브랜드 프로필이 로드되지 않은 상태에서는 어떤 작업도 시작하지 않음

---

## 글 유형 자동 판별 규칙

- 브랜드 프로필에 "글 유형 구분"이 있는 경우:
  - 입력된 키워드를 브랜드 프로필의 "우선 공략 키워드" 유형별 목록과 대조
  - 매장용 키워드에 가까우면 → `store` 유형 적용 (해당 CTA, 글 구조)
  - 단체주문/케이터링 키워드에 가까우면 → `catering` 유형 적용 (해당 CTA, 글 구조)
  - 판별 불가 시 → 사용자에게 확인
- 브랜드 프로필에 글 유형 구분이 없으면 → 단일 유형으로 처리

---

## 실제 고객 후기 활용 규칙

- 브랜드 프로필에 "실제 고객 후기 기반 만족 포인트" 섹션이 있으면:
  - 블로그 글 작성 시 해당 포인트를 자연스럽게 근거 자료로 활용
  - 후기 원문 그대로 복붙 금지, 매장 소개형/정보 제공형으로 재구성
  - "후기에 따르면~" 같은 인용 표현 금지 → 자연스러운 매장 설명으로 녹이기
- 브랜드 프로필에 "브랜드 핵심 스토리" 섹션이 있으면:
  - 글의 설득력과 신뢰도를 높이는 근거로 활용
  - 매번 똑같이 넣지 말고, 해당 글의 주제에 맞을 때 자연스럽게 녹이기

---

## 전문가 팀 파이프라인 (agents/ 폴더 참조)

- **STEP 0**: 키워드 전문가 → API로 검색량 조회
- **STEP 1**: 키워드 전문가 → 키워드 리포트 (대표 1개 + 서브 3~5개)
- **STEP 2**: 경쟁 분석가 → 상위 5개 글 텍스트마이닝
- **STEP 3**: 콘텐츠 기획자 → 제목 3안 + 도입부 + 소제목
- **STEP 4**: 카피라이터 → 본문 작성 (브랜드 톤 적용, 1,500~3,000자)
- **STEP 5**: SEO 전문가 → 키워드 배치 검수 + 메타 디스크립션
- **STEP 6**: 교정 편집자 → 맞춤법 + AI투 제거
- **STEP 7**: 팀장 → 최종 검수 체크리스트 점수화 (80점 이상 통과)

---

## 핵심 작업 규칙

- **글 길이**: 1,500~3,000자 (공백 포함)
- **키워드**: 제목에 메인 1개, 본문에 서브 2~3개
- **키워드 반복 상한**: 메인 총 5회 이내 (초과 시 동의어 대체)
- **문체**: 브랜드 프로필 `톤앤매너` 필드에 따름
- **소제목**: H2 3~5개
- **결론**: 반드시 브랜드 프로필의 글 유형에 맞는 CTA 포함
- **브랜드 프로필에 없는 정보는 절대 지어내지 않음**

---

## 글 구조

1. **도입부**: 공감/질문/놀라운 사실로 시작 (3~4줄, 첫 3줄이 홈피드 승부)
2. **본문**: 소제목 3~5개로 구분
3. **정리**: 핵심 3줄 요약
4. **CTA**: 글 유형에 맞는 CTA 적용

---

## 출력 규격

- **형식**: markdown (.md), UTF-8
- **블로그 글**: `./output/[brand-slug]/blog/YYYYMMDD_[키워드슬러그].md`
- **기획안**: `./output/[brand-slug]/plans/`
- **감사 보고서**: `./output/[brand-slug]/audits/`

## 다양성 규칙 (매번 다른 주제/구조 필수)

- 같은 글 구조 2회 연속 사용 금지
- 같은 소제목/주제 반복 금지: 이전에 작성한 글의 소제목과 겹치면 안 됨
- 시작 방식 순환: 질문형 → 공감형 → 놀라운사실형 → 이야기형 → 상황묘사형
- 같은 메뉴를 메인으로 다루는 글을 연속 작성 금지
- 매 글마다 다른 "절입 각도"를 사용:

  | 각도 | 설명 |
  |------|------|
  | 메뉴 중심 | 특정 메뉴 1개 깊이 소개 |
  | 상황 중심 | 소개팅, 혼밥, 데이트, 재택근무 등 |
  | 공간 중심 | 테라스, 인테리어, 포토존 등 |
  | 시간대 중심 | 주말 브런치, 평일 저녁, 점심 등 |
  | 스토리 중심 | 운영 스토리, 쉐프 철학 등 |
  | 시즌 중심 | 벚꽃 시즌, 여름 테라스, 크리스마스 등 |
  | 고객 유형 중심 | 1인 고객, 커플, 소개팅, 단체 등 |

- **batch 생성 시**: 키워드 목록 전체를 먼저 확인하고, 각 키워드마다 다른 절입 각도를 미리 배정한 후 작성 시작
- `output/[brand-slug]/blog/` 폴더의 기존 파일 제목을 확인하고, 겹치는 구조/각도 회피

---

## 금지 사항 (모든 브랜드 공통)

- 과도한 키워드 반복 (스팸 판정 위험)
- 근거 없는 수치/통계 인용
- 브랜드 프로필에 없는 정보 지어내기
- 타 브랜드 비하
- 브랜드 프로필 `금지 표현` 필드에 명시된 표현
- AI 특유 딱딱한 표현 (브랜드 톤이 허용하지 않는 한)

---

## 커맨드 규칙

- 모든 커맨드는 파이프(`|`) 기준으로 왼쪽=브랜드, 오른쪽=키워드/파일경로
- 커맨드 상세는 `.claude/commands/` 폴더의 각 `.md` 파일 참조
